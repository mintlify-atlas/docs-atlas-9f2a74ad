# Installation

Get duv'Assistant up and running in your Visual Studio Code environment.

## Prerequisites

Before installing duv'Assistant, ensure you have:

- **Visual Studio Code** version `1.96.0` or higher
- **OpenAI API Key** (required for AI completions)
- Active internet connection for API requests

<Note>
You can check your VS Code version by going to **Help > About** in the menu bar.
</Note>

## Installation Steps

<Steps>
  <Step title="Open VS Code Extensions">
    Launch Visual Studio Code and open the Extensions view by:
    - Clicking the Extensions icon in the Activity Bar (left sidebar)
    - Or pressing `Ctrl+Shift+X` (Windows/Linux) or `Cmd+Shift+X` (macOS)
  </Step>

  <Step title="Search for duv'Assistant">
    In the Extensions search bar, type `duv'Asisstant` to find the extension.
    
    <Note>
    Note the spelling: "duv'Asisstant" with the specific capitalization and apostrophe.
    </Note>
  </Step>

  <Step title="Install the Extension">
    Click the **Install** button next to the duv'Asisstant extension.
    
    The extension will download and install automatically. Once complete, you'll see a notification confirming the installation.
  </Step>

  <Step title="Reload VS Code (if prompted)">
    Some installations may require reloading VS Code. If prompted, click **Reload** to activate the extension.
  </Step>
</Steps>

## Development Setup

<Warning>
**Important**: This extension is currently in early development. Configuration requires building from source, as there is no VS Code settings UI integration yet.
</Warning>

### For Developers: Building from Source

To configure the extension with your own API key, you'll need to build it from source:

<Steps>
  <Step title="Clone the Repository">
    Clone the extension source code:
    
    ```bash
    git clone https://github.com/DUVAN100/duv-asisstant.git
    cd duv-asisstant
    npm install
    ```
  </Step>

  <Step title="Get Your OpenAI API Key">
    If you don't have an OpenAI API key:
    
    1. Visit [OpenAI Platform](https://platform.openai.com/)
    2. Sign up or log in to your account
    3. Navigate to **API Keys** in your account settings
    4. Click **Create new secret key**
    5. Copy the generated key
    
    <Warning>
    Keep your API key secure and never commit it to version control.
    </Warning>
  </Step>

  <Step title="Configure API Key">
    Edit `src/config/settings.ts` and replace the placeholder with your API key:
    
    ```typescript
    export const settings = {
        openAI: {
            apiKey: 'sk-your-actual-api-key-here',
            model: 'code-davinci-002',
        },
        supportedLanguages: ['javascript', 'typescript', 'python'],
    };
    ```
  </Step>

  <Step title="Build and Run">
    Build the extension and run it in development mode:
    
    ```bash
    npm run compile
    ```
    
    Then press `F5` in VS Code to launch the extension in a new Extension Development Host window.
  </Step>
</Steps>

<Note>
Future versions will include a proper VS Code settings UI for easier configuration without rebuilding.
</Note>

## Verify Installation

Confirm that duv'Assistant is working correctly:

<Steps>
  <Step title="Check Extension Status">
    1. Open the Extensions view (`Ctrl+Shift+X` / `Cmd+Shift+X`)
    2. Search for "duv'Asisstant"
    3. Verify that it shows as **Enabled**
  </Step>

  <Step title="Look for Activation Message">
    When you open a JavaScript, TypeScript, or Python file, you should see a notification message:
    
    ```
    Extensión de autocompletado activada.
    ```
    
    This confirms that the extension has activated successfully for supported languages.
  </Step>

  <Step title="Test Snippet Completion">
    Create a new `.js` file and type `console-log` to verify that snippet completions are working. You should see the completion suggestion appear in the IntelliSense menu.
  </Step>
</Steps>

## Supported Languages

By default, duv'Assistant activates for the following languages:

- JavaScript (`.js`)
- TypeScript (`.ts`, `.tsx`)
- Python (`.py`)

<Note>
To add more languages, modify the `supportedLanguages` array in `src/config/settings.ts` when building from source. See the [Configuration](/guides/configuration) guide for details.
</Note>

## Troubleshooting

### Extension Not Activating

If the extension doesn't activate:

1. Verify your VS Code version meets the minimum requirement (1.96.0+)
2. Check that you're working with a supported file type
3. Try reloading VS Code with `Developer: Reload Window` from the Command Palette

### No AI Completions Appearing

If AI completions aren't working:

1. Verify your OpenAI API key is correctly configured in `src/config/settings.ts` (if building from source)
2. Check your internet connection
3. Review the VS Code **Output** panel for error messages
4. Ensure you have available credits in your OpenAI account
5. Note: The AI completion feature is in development and may require additional API endpoint configuration

### Snippets Not Showing

If snippets aren't appearing:

1. Ensure you're typing the exact snippet trigger (`for-loop`, `console-log`)
2. Check that IntelliSense is enabled in VS Code settings
3. Try pressing `Ctrl+Space` to manually trigger completions
